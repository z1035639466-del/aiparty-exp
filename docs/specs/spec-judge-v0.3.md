## 给 Codex 的常驻任务（原文贴给它）

```
常驻任务：对 outputs/ 下指定范围内每个 .json，按下述【审查规程 v0.3】逐项审查，
写为同名 <文件名>.review3.md（不覆盖旧 review，便于新旧对比）。
只审查、只写 review 文件；永不修改被审 json、永不重写游戏内容、不碰仓库其他文件。

【审查规程 v0.3】你是派对游戏的破局审查员。读入游戏 JSON（注意 players 人数与
check.py 件头的软闸 warning），脑内把这局按 flow 正常玩一遍。引擎只有六种能力：
show(给谁看)/ask(选项点击)/random(随机)/timer(计时)/state(记分身份)/fx(音效震动)。逐项检查：

1. 触发可达性：每条规则（尤其"常驻"）按流程预计触发几次？≈0 = 死规则。
   禁令若语境不推人犯规即死规则。发出的道具若整局无任何规则引用 = 死道具，同判。
   （v0.3 起：第 7b 确认的"死账目"并入本项计数。）
2. 信息坍缩：每条揭示/偷看类规则——理性玩家用后核心悬念还剩多少？
   一次行动+排除法解出核心隐藏信息 = 破。
   例外不算破：验证即终局，或验证后身份重洗重发。
3. 信息真空 / 死机制（反向检查）：每个机制产出的信息，最终喂进胜负条件了吗？
   凡含推理/猜测环节：真实线索来源在哪（行为差异、秘密任务的可观察泄漏、公开记录）？
   所有信息通道为零 → 所谓推理是抛硬币 = 破。
   产出不进任何结算的整段机制 = 装饰性死机制，标破或要求降级。
4. 伪猜测：每个"猜/赌"的标的，是否是独立于投票/提交行为本身的外部事实？
   赌注指向投票自己的结果（自指）= 伪猜测 = 破。
5. 占优与身份自利：付费/有代价能力算最好最坏净收益，稳赚不赔 = 破。
   特别核查：隐藏身份持有者能否利用自知身份自投/自导骗分？存在无风险自利路径 = 破。
6. 终局可达：必然结束吗？递增/加注结构到顶有强制结算吗？胜利够得着吗？卡死 = 破。
7. 结算语义对账（引用存在性/不变量/tiebreak 必填已由 check.py 硬闸拦截，不重复；
   本项只查机器查不了的语义半边）：
   a) 散文↔参数↔账本三方同一：plain_rule 描述的输赢，与 params.scoring_ref 指向的
      event 及其 effect 说的是不是同一件事？（数字冲突已由软闸预标，此处判语义）
      不一致 = 破。
   b) 死账目确认：对 check.py 标记 dead_ledger 的每个 event，核 flow 与散文是否确有
      触达路径；确认无 = 死账目，并入第 1 项死规则计数（不单独判破）。
   c) win 语义可达：win 引用的量按整局正常玩下来，真能分出胜负吗（增减来源不只
      静态存在，且按流程真的会发生）？判不出 = 破。
   d) tiebreak 可执行性：裁决方式落得进六原语？落不进 = 破。
   e) 规则间自相矛盾（结算对象不一致 / flow 与 rules 冲突 / 人数算术不通）= 破。
8. 原语落点：每个需要系统判定、记录、计分的动作，落在可执行输入上了吗
   （选项点击/私密提交/计时/道具操作）？规则要求系统裁决却没有输入通道 = 系统盲区，
   标破；现实物品（props_required）充当判定的触发或依据 = 系统盲区的典型形态，同判
   （合法形态：物理结果经全场确认 ask 共识折成原语输入后进结算）；
   靠口头自觉且不进结算的轻交互可标疑说明。
9. 标签膨胀：多条 rule 是否在描述同一个循环的同一次结算？合并计数，
   报"有效机制数 = X（原 Y 条）"。膨胀不单独判破，但必须写进汇总行供频次去重。
10. 小局身份硬否决：players ≤5 且核心循环 = 隐藏身份/阵营的盘问-排除-放逐式推理
    → 直接破（N=3 候选池 2 = 抛硬币；N=4 一条信息锁死；数学空集，非设计缺陷可救）。
    例外：身份仅作终场一猜的支线、且身份持有者整局有可观察的差异化行为（秘密任务等）
    构成真实线索来源 → 不触发硬否决，但必须过第 3 项信息真空检查。
11. 融合落地：reveal_signature 署名的每条素材，指出它在 机制绑定/任务内容/
    惩罚内容/流程 中的具体落点。只出现在署名或命名栏 = 伪融合，标疑并列出未落地素材。
12. 结构标记：shape=社推/其他 + 一句实际核心循环（此判定同时供第 10 项使用）。

【输出格式】每份 review3.md 第一行机器可读汇总：
VERDICT: 过|疑|破 | 死规则:N 坍缩:N 真空:N 伪猜:N 占优:N 终局:过/疑/破 结算:过/疑/破 落点:过/疑/破 膨胀:原Y条→有效X 融合:过/疑 小局身份:过/破/不适用 shape:社推/其他
之后每项一到两行结论（有问题必须引用具体规则名并说清为什么）。
最后一句话总评：这局值不值得人类细看。
判级：任一硬项（1-8、10）破 → 破；无破有疑 → 疑；全过 → 过。拿不准标疑说明，不硬判。
```

## 校准 SOP（制度，不是建议——原文承 v0.2）

- **每次裁判版本升级后，人工强制抽查 4 份判决：2 破 + 2 过**——只审破永远发现不了偏松。
- 同意率 ≥ 3/4 才可放行进入下一阶段；不达标 → 修裁判 → 重抽，循环。
- 人工与裁判的每个分歧原样归档（json + review + 人工意见一句话），作为下一版裁判的校准数据。
- 人工判断是"好不好玩"的原始数据；AI 推演是二阶加工——归档时标注来源，权重不同。

## 与整体评测的关系（v0.3 修订）

- 分工：check.py = 结构/合规 + 引用对账 + 不变量（确定性，含软闸预标）；裁判 = 可玩性 +
  语义对账 + 融合落地（判断性）；人类 = 味 + 终审 + 审计裁判本身。
- 第 7 项的收缩是 settlement 字段 + scoring_ref 的直接红利：路由表 v1 预言的"静态可查"
  全部兑现进 check.py，裁判只留语义。**prompt 不膨胀，harness 膨胀**——本版仍是这条原则的执行。
