# 道具库 v0 · 定稿（电子桌面道具规格）

> 配套：《游戏底层设计 v0.2》(总纲)、冲刺一/二开发文档、《DM Skill · 开局生成》(运行 prompt)。
> 本文档是"道具层"的规格，和"规则层(6 原语)"并列，是 AI 现制游戏时可调用的另一半地基。
> **本文档标志设计阶段收口。定稿之后进入建设与"通电"。**

---

## 0. 术语（先统一，全项目照此）

aiparty **不用「许愿」这个词**（那词不准确，且与另一独立项目"许愿时钟"撞名，避免污染）。全线改用：

- **口味**：偏好滑块（烈度/节奏/亲密度…）。量化的调味旋钮。
- **素材**：玩家提供的元素、梗、热点。要被融进游戏的原料。
- **配局**：开局前收集口味与素材、由 AI 现制这一局的整个环节。

产品核心动作的准确表述：**玩家提供口味与素材 → AI 据此配局**。不是"许愿→满足"。

## 1. 四根地基（1.1–1.3 数字道具 · 1.4 现实物品）

### 1.1 权限边界：造道具是真人的权限，用道具是 AI 的权限

- **平台造的道具**（骰盅、左轮、身份牌…）：我们真人写死在程序里。
- **创作者提供的道具**：第三方真人做的，进库前过审。
- **AI 在两者上永远只有一种权限**：从已存在的库里 **挑 / 发 / 命名 / 配文案**。
  **它不能造一件新道具，也不能改一件道具的机制。**

**为什么这条不能松**：白名单安全模型（validator/contentFilter/patchRouter）靠"只有库里这些合法、其余一律 forbidden"才成立。一旦 AI 能造道具，就能造任意效果，安全闸没有"合法清单"可比对，整套就塌了。所以"道具原语只真人有权限"不是权限设置，是**安全模型的前提**。架构上：道具原语 = 和"6 原语"并列的白名单地基，**真人可写、AI 只读只调**。与 schema"不新增原语"同一条纪律。

### 1.2 筛选准则：只做"补现实短板"的东西

大家同处一室，现实已免费给了：真实座位、左右顺序、谁挨着谁、面对面、递东西、氛围。**app 不重复这些**，只补现实做不好的三样：

| 短板 | 含义 |
|---|---|
| **隐藏** | 该藏的信息（只有你看得到 / 别人看你看不到） |
| **随机** | 公平且不可篡改的随机 |
| **记忆·裁判** | 记住谁该谁了 / 分数 / 阶段，中立不偏袒 |

一个东西中不中这三条，决定它进不进库。纯复刻现实已有的（座位、距离），不做或轻做。

### 1.3 可见性引擎：所有道具插在同一个开关上

每件道具下发时带一个**信息朝向**。这一个开关就是"信息反转"引擎：

- `自己看`（私密）
- `额头`（**别人看得到、你自己看不到**）
- `全场公开`

**同一件道具换个可见性就是另一种玩法**（一副牌：自己看=手牌 / 额头=谁是卧底 / 公开=明牌堆）。所以库不用做很多件，把可见性用足即可。这一档也是"现实做不到、app 才有价值"的地方。

### 1.4 现实物品边界：给指称不给感知（正典句）

1.1–1.3 管的是**数字道具**（平台写死、走 14 库白名单、插可见性引擎、schema 顶层 `props_dealt` 承载）。**现实物品**（线下真实道具：瓶子、筷子、花生、平板…，schema 顶层 `props_required` 承载）走**另一条轨**，其边界由一句正典锁死：

> **给指称不给感知　+　ask 共识唯一通道　+　属性即证词。**

三层拆解：

- **给指称不给感知**：系统只把现实物品当**指称**（"那瓶酒"）透传进任务/挑战/惩罚文案，**永不感知**它的物理状态。它永不进任何机器槽（`verify_source` / `draw_from` / `reveal_of` / `on_*`…）——判定永远走六原语（`design-layer-v2.0.md` §2 C7：props_required 命名空间进机器槽 = 硬闸）。
- **ask 共识唯一通道**：物理事件（谁做到了、谁先完成、抢到没抢到）进结算的**唯一合法通道**是全场共识确认（tap / 提交 / 计时 / 证词对账），从不是物理事件本身。"谁先抢到哑铃"没有通道 = 系统盲区，僭越线不动。
- **属性即证词**：现实物品的属性（这瓶多少度、多少钱）只有**配局时明确上传的**才能作谜底/判据；DM **不得替现实物品发明属性**（发明 = 对不存在证词的引用，信息真空的现实版）。

**挑战媒介五式**（判定通道全部落在 MVP 7 机制内，编译器 v0 原生支持、零扩容）：

| 式 | 挑战形态 | 判定通道 | 现实物品角色 |
|---|---|---|---|
| 完成式 | 做到/没做到（单手开瓶、平板支撑） | `判定` source=consensus，on 过/不过 | 挑战标的 |
| 计时式 | 限时做完（30 秒夹花生） | `限时` seconds + 完成共识 tap；超时走 on_timeout | 标的 + 计时皮 |
| 竞猜式 | 猜物品属性（多少度/多少钱） | `同时提交` 私密提交，谜底=配局上传的属性证词 | 谜面 + 谜底来源 |
| 声称式 | 我声称我能……（吹牛+验证） | `声明质疑` verify_source=judge（全场共识验证），on_liar / on_false_accuse | 声称标的 |
| 递增式 | 再举十秒还是收手保分 | `续押喊停` bust 由自报/共识 tap 折原语；cap/on_cap 不变量照焊 | 耐力/胆量标的 |

结算侧统一由 `加减分` / `惩罚三档` 承接（`scoring_ref` 对账照常）。物理动作类挑战（玻璃器皿/重物/竞速饮酒）倾向**黑名单**，safety_note 联动、尺度分级管边界——语料已标"应淘汰"的先例（一気飲み / Boat Race 类）照禁。

> 字段键 `props_required` 由 `design-layer-v2.0.md`（§0/§4，C6）冻结；本节为其在道具层的正典边界。

## 2. 风味层 / 机制层（个性化的核心引擎）

个性化不靠"复读玩家说的话"，靠**AI 懂那个梗背后的整个世界、把它编进玩法**。为兼顾创造性与可实现性，把"元素"劈成两层，走相反的规则：

- **风味层（无限）**：皮、名字、文案、术语、氛围、惩罚的**描述**。AI 尽情把"重金属"炸开成整个世界（mosh pit / 双踩 / solo / 砸吉他…）。**只影响观感，不改任何数值和流程。** 这层放开。
- **机制层（有限）**：真正决定"怎么玩"的东西——回合、目标、可见性、结算、触发、用哪几件道具。**只能从固定库挑，一个新机制都不能造。**

**魔法发生在"绑定"上**：AI 把联想出的风味子元素，**映射到已有机制**，穿皮 + 起名 + 配文案。例：mosh pit → 绑`转移`(把坏结果甩给邻座)，叫"入坑冲撞"；solo → 绑`限时对决`；砸吉他 → 绑`加倍惩罚`。**风味在长，机制原地不动。**

**由此立两条硬约束**（已写进 DM skill）：

> **约束 A（绑定）**：每个风味元素必须绑定到库里一个已有机制。**绑不上已有机制的风味，只能进文案，不能进玩法。**
>
> **约束 B（白话）**：每个风味落成规则时，必须同时给①风味皮 和 ②一句不含任何梗的白话机制说明，让**完全不懂这个梗的人也能独立看懂该做什么**。看不懂机制说明的风味 = 作废。

约束 B 的理由：一桌里**总有人不懂那个梗**。风味是氛围的糖衣，不是理解规则的前提。AI 越个性化，越不能把不懂的人排除在外，也防止 AI 把规则藏进黑话。

> 体验开关（**非 v0 必做，先记**）：**风味浓度**——想沉浸的看风味名为主，想快点搞懂的一键切纯白话规则。懂的和不懂的各取所需。

## 3. 梗 / 元素的处理链路

道具走白名单，**梗走"素材注入 + 出口过滤"，且由玩家自己带进来**。手法：

1. **梗由玩家在配局环节自己提供，不是 AI 去"回忆"名人梗。** 这是命门。让 AI 凭记忆编"某名人的梗"，就是在赌真假、赌会不会诽谤——不可控。让玩家自己填他们圈内的梗/爱的台词，拿到的是**用户自愿、指向其私人圈子**的素材，责任、真实性、语境全对。
2. **AI 只做"缝合"，不做"发明"。** 拿玩家给的素材，硬融合进规则皮和揭幕文案 + 署名。融合低风险(素材来源可信、署名标出处)，发明高风险(内容 AI 编、来源不明)。**只融合、不发明。**
3. **出口全过闸。** 玩家带进来的梗照样过 contentFilter——玩家自己也可能给越界内容。**来源可信 ≠ 内容免检。**
4. **私人梗只在本桌用、不外传、不沉淀跨局。** 顺带把版权/名誉风险摁到最小。

**对 AI 的硬禁令**（写进 DM skill）：

> AI 可围绕主题**自由联想风味**(氛围/术语/名字)，但**禁止自行搜索记忆或编造任何具体的真人言论、点名名人的梗、有版权的歌词台词**。任何具体真人/IP 内容，必须来自玩家素材。

**主动搜索附属热梗**（AI 去联网/发散找相关热点再编进游戏）——想要，但 **v0 预留、不实现**。理由：这一步的成本/时延/味道要靠 agent桌跑批才测得出，且失控风险高。**v0 先只做"绑定"这半边**（AI 用玩家给的 + 模型自带联想去贴皮配味），已能做出"重金属味的 mosh pit 冲撞"这种效果。接口与 skill 段落先留槽、标注不实现。

## 4. 技能卡（电游式特殊道具）——预留，v0 不实现

"按口味发特殊道具"(复活卡那种)——想法保留，但收窄且缓做。

**收窄**：不让 AI 造效果，把技能卡也做成**固定效果小库**，AI 只能从中挑、发、命名、配文案。所有效果原子都是对引擎**已有操作的修饰**，不是新机制：
`免死 / 反弹 / 加倍 / 透视 / 点名 / 跳过 / 交换`。
（`透视` = 临时把某道具可见性对你翻成"自己看"，仍插在可见性引擎上——技能卡不是独立新系统。）

**接口收口**：只加**一个新道具类型 `技能卡`**，效果只能选上面的原子；只有它的"发卡"操作带一个可选 `偏好` 钩子（参考某人口味决定发谁、叫啥）。**口味只影响"发给谁 + 皮和名字"，永远碰不到机制。** 口子就这一个。

**平衡取向（已定）**：**故意不对称**——烈度拉满的人拿高风险高回报的卡，低烈度的人拿保命卡。接受"不对称就是设计的一部分"。

**开关（已定）**：整个个性化特殊道具带 **开关，默认关**。

**建设状态（已定）**：**v0 不实现，接口预留、标注**。理由：不对称技能卡叠在一个"还没验证好不好玩"的核心上，又引入不对称平衡 + AI 按口味挑卡两个难题，二者都要靠真人数据才调得动；且默认关，无上线紧迫性。**设计缝、先不填肉。**

## 6. 组合示例（同一库，挑几件就是不同的局）

- **炸盅之夜**：骰盅(自己看) + 额头身份牌(反转) + 记分板。诈唬 + 身份反转。
- **今夜谁扣扳机**：虚拟左轮(公开) + 命运转盘(转惩罚) + 沙漏。纯张力。
- **烫手真心话**：定时炸弹(隐藏倒计时) + 密语卡(私发任务) + 抽签筒(定顺序)。
- **谁是内鬼**：密语卡(发词) + 匿名投票器(放逐) + 记分板。社交推理。

**核心循环**：玩家给口味+素材 → AI 挑道具、配可见性、绑风味、编规则 → 下发到各人手上 → 玩 → 吐槽 → AI 秒改 → 揭幕(硬融合+署名)。

## 7. 建设优先级 + 通电

**别一次做全 14 件。** 按"覆盖最多玩法 / 最快能跑一局"：

- **信息反转核心（先做，这是护城河）**：身份牌、骰盅、密语卡、盲盒——四件占满三档可见性。
- **张力担当（好玩上限，先做一两件）**：虚拟左轮 或 定时炸弹。
- **骨架**：记分板、沙漏。
- **地基积木**：扑克牌、骰子。

做每件时，一次要做对的是它挂在**可见性引擎**上的那个接口——可见性对了，后面加道具都是往同一插槽插。

**下一步不是再加设计，是通电**（把"我觉得"变成"我看到"）：
1. 本文档定稿 → **停止设计。**
2. 写 DM skill 最小版（已产出开局生成 prompt） + 挑 2–3 件道具 + 3–4 个机制原子。
3. **agent桌跑通哪怕一局**：拿到第一个"AI 生成 → 过校验 → 能玩"的闭环。
4. 拉几个真人（**含一个不懂梗的**）真机玩一局。

到这一步才第一次有"过校验通过率 / 延迟 / 成本 / 味够不够 / 白话清不清"的真数据。**"最好的架构"在通电之后才浮现，不在通电之前。**

## 8. 调研提炼（v1.1 增量，来源：300+ 款全球游戏 / 36 母机制家族）

四份调研（线下六大区 + 八国补采 + 六国二轮 + Steam 近5年）证实与新增如下，已全部并入《DM Skill》v1.1：

**三层架构确认**：调研自身就是按引擎真实六原语（show/ask/random/timer/state/fx）做的映射——证实分层为：**六原语 = 指令集(ISA)；道具原语 = 标准库(由六原语组装)；游戏脚本 = 纯数据**。可见性三档正是 show 的参数空间；花火(自己的牌只有别人看得见)证明"扑克牌+额头档"零新增即可表达——**可见性用足、库不用大**这条被外部数据独立验证。

**结果机制 7→13**（每条均有 5+ 款跨文化游戏背书）：新增 声明质疑（叫价诈唬家族：大话骰/Coup/骗子牌/过关申报）、同时提交（剪刀石头布/继续撤退/默契比对/预测彩池——数字化后杜绝偷看，调研标注"A级：比实体更公正"）、续押喊停（Skull/印加宝藏式贪心博弈）、受限沟通（一词一数/是否无关三态/同义词，Codenames/情景谜题/二十问家族）、判定（系统按规则表达式自动判对错——数字接龙/爬梯出牌/猜码反馈家族，调研称"ROI 最高"；判不了的两个替代判源：全场共识确认、AI 语义仲裁且可被全场一键推翻）、传递链（传话失真家族：私密逐传+全链留痕+终局回放）。另增规则属性 **常驻**（整场后台生效、举报驱动，调研中唯一的"背景常驻式"结构）。

**修正**：身份牌默认档从"额头"改为"自己看/额头双档"——社交推理家族（狼人/卧底/Coup，调研认定"天花板最高"）全部用"自己看"，额头档留给猜自己式。匿名投票器升级为**同时提交器**（投票只是它的特例）。

**AI 解锁 B 级**：调研在无 AI 前提下把"自由填词/编造答案"类评为 B（语义仲裁需人工）。本产品的 AI 语义仲裁（可被共识推翻）把这一整层拉近 A——词语接龙、瞎编答案骗人（Fibbage 式）、藏头诗评比等家族由此入库可用。

**安全护栏（升为铁律）**：大冒险/国王指令家族的跨国教训（日本王様ゲーム因骚扰被企业限制）→ 指向个人的任务必带拒绝权、拒绝代价温和不羞辱；饮酒节奏家族 → 系统只发节拍永不逼量、惩罚给非饮酒替代项；题库按场合分级（家庭/职场/密友）。

**明确不迁移（筛子被独立验证）**：投掷精度、身体追逐接触、灌酒竞速三大 C 级家族，调研结论与本规格铁律 5 完全一致——乐趣在物理本身的，保留为"人类剧场"，app 只出题/计时/收评判。

**引擎表达力测试集（附录 B 第 8 条的具体化）**：以 36 母机制家族为测试集，**A 级家族（约 25 个）为必测项**——每个家族问"能否不新增原语纯用 schema 表达"。其中 吃墩叫牌合约(22)/摸牌组合成型(27)/骰子棋盘(28) 三个家族规则引擎工程量大，列为**表达力压力测试项**（测 schema 表不表达得了，不代表 v0 要实现）。

## 9. 运行环境与端划分

**权威的东西在服务端，呈现的东西在客户端。**

- **服务端（逻辑库）**：道具的状态与逻辑——点数、额头牌归属、炸弹随机倒计时、左轮膛位——全部服务端权威，随机一律走 rngSeed。理由：客户端不可信；道具库的全部价值（隐藏/随机/裁判）只有可信服务端做得到。
- **客户端（素材库）**：道具的皮——模型/贴图/音效/动画——预打包在客户端，渲染零等待。AI 全程只报道具 ID 与参数，永不发素材、永不碰客户端。
- **客户端只渲染引擎状态，永不渲染 AI 文本**（与附录 B 执行层一致）。
- **AI 只在接缝**：配局生成/秒改/问询/续杯发生在开局前、吐槽后、回合间隙，秒级等待可接受；游戏内快动作（摇/扣/投）走"本地乐观渲染 + 服务端权威确认"，永不经过 AI。
- **交互档（渐进增强，永不按端切道具库）**：每件道具声明三层——基线 = 点击+视觉+音效（全平台保底，任何道具仅靠基线即完整可玩）；增强 = 震动/摇一摇体感（原生 app 或安卓浏览器点亮）；退化 = iOS 浏览器上增强层优雅降级回基线，fx 震动标签必须有视觉等价物。**把一种感官锁在 app 后面，永远不把一件道具锁在 app 后面。** H5 零安装是通用基线（派对现场不可能让客人先装 app），原生 app 是以后给熟客的增强层。

## 10. 新道具入库程序（通用，一次约一小时机器时间 + 人工抽查）

> 阶段 1 的三周是造验证机器的一次性成本；此后每件新道具走以下增量程序，不重走全程。
> 失效类检查（死规则/占优/信息坍缩/平局/终局）全部写在类级、与具体道具无关，对新道具自动生效。

1. **立据**：确认哪个预注册触发器亮了——烂例双诊断"引擎不够"格出现重复形状 / 多样性跑批塌缩 / 表达力测试测出缺口。没有触发器不入库。
2. **定义**：按本规格模板写清——名称、对应现实道具、补哪个短板(隐藏/随机/记忆裁判)、可见性档、操作集、交互档(基线点击→增强震动体感→iOS 退化)。
3. **数据更新**：白名单 +1、DM skill 固定库表 +1 行；若带新的人数/口味约束，适配表 +1 行。
4. **探针输入 ×1**：造一套素材天然召唤它的配局输入，并入 golden set（input_D 模式）。
5. **回归批**：golden set 全量 + 探针，过 check.py + 裁判；VERDICT 无新破、旧类零复发才放行。
6. **分布确认**：频次表看它被用了没有、有没有虹吸其他道具（密语卡教训）。
7. **引擎侧（生产期）**：执行器 + 可见性钩子 + 客户端皮，真人编写（权限模型），过附录 B 执行层六站。
