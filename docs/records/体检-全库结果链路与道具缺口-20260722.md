## 结果来源五型(体检的尺子)

在"结算/判定时刻,局长凭什么工具拿到'谁输/谁进结算'"这个问题上分五型:

- **A 系统自动**:线上道具后台有数,不问、自动走。现有面:`prop.dice_cup`(骰盅对账信道,仅局长可见)、`random.dice/pick/int`(当众公摇/随机点人/私密摇)。
- **B 判定按钮/仪器**:终点单点交互或感知判定。现有面:`duel.start`(快枪手拔,毫秒系统判)、`judge.photo/audio`(视觉/听觉裁判)、大话骰开牌按钮(玩家端 roll 事件)。
- **C 问询回账**:实体道具/线下动作,结果玩家玩出来,局长 `ask` 问一嘴或玩家 `say(to=局长)` 上报,`state.note` 入账。
- **D 共识表决**:无客观事实,靠全桌投票/多数认(`ask` + options 现成)。
- **E 断链**:以上四路都走不通,或走 C 会明显破坏体验(高频每轮都要问、结果复杂一句话说不清、感知无观察者)——这是要修的。

## 四、E 型病灶清单(逐条)

### 硬断链(现面四路皆不通)

**E-1 全员闭眼型 · 无观察者**(6 条 + 卡 pat-v1-53):谁都闭眼→AI 局长看不见、桌上也看不见,"先睁眼"无从取源。这是比大话骰更彻底的断链(大话骰后台还有点数,这个连数都没有)。
- xhs-01315 / xhs-01702 / xhs-01787 / xhs-01958 / xhs-01988 / xhs-03728:大喊"都闭眼",谁先睁眼谁喝。
- 修法:转成**抢按/反应仪**(系统随机亮灯,先/后拍屏定胜负,duel 的多人泛化),把感知搬进系统。

**E-2 有状态虚拟牌桌**(约 25 条 + 卡 pat-t1-06):多轮手牌/出牌/比大小,状态跨轮持久,一句 ask 说不清、也无牌库发牌。
- xhs-00134 猜牌决斗 / xhs-00135 田忌赛马 / xhs-00136 皮诺曹 / xhs-00137 斗地主 / xhs-00140 钓鱼 / xhs-00142 千瞳眼 / xhs-00145 骗子酒馆 / xhs-01033 变色龙 / xhs-02690 七星龙珠 / xhs-02694 谁最慢 / xhs-02695 金字塔 / xhs-02698 数八张 / xhs-02699 比7 / xhs-03452 小姐牌 / xhs-03453 十点半 / xhs-03450 炸弹接龙 / xhs-03487 抽鬼牌 / xhs-04100 比大小 等。
- 修法:**虚拟牌库/牌桌原语**(系统发牌到私密信道、玩家端翻/出牌、后台记状态,A 型闭环)。这是解锁量最大的单件。

**E-3 精确计数听觉**(2–3 条):数掌声/听掌声报次数,人耳/AI 都难精确计数裁定。
- xhs-01166 听掌声说几次 / xhs-02854 数掌声数错者喝。
- 修法:系统播放固定拍数(音频资产)+ 玩家报数比对,或直接归入低优先冷门。

## 五、道具原语缺口排行(按解锁量排序)

实现量级参照 `prop.dice_cup` 骨架:**发(局长发放,桌上公开)→ 玩家动作(玩家端自己玩)→ 后台结果(对账信道,仅局长可见)→ 对账/公开(局长凭信道走后续)**。

| 排名 | 候选原语 | 型 | 解锁量(卡 / high 原子) | 净新增(现面外的能力) | 实现量级 |
|---|---|---|---|---|---|
| 1 | **虚拟牌库/牌桌 `prop.deck`** | A/B | 卡 1(七星龙珠)+ 55 原子(43 完整玩法,内 25 硬断链牌桌) | 极高:现面 0 支持,25 条纯断链只此一路 | 大:需发牌到私密信道 + 玩家端翻/出牌事件 + 后台手牌状态机 + 开牌对账。比骰盅重(多轮持久状态) |
| 2 | **抢按/多人反应仪 `duel` 泛化** | B | 卡 6(t1-01·t1-05·v1-28·v1-37·v1-50·v1-53)+ 58 原子(48 完整玩法) | 高:闭眼型(6 硬断链)唯一解;把反应/急停/竞速一大簇从 C 退化升到系统毫秒判 | 中:duel.start 已有毫秒判骨架,泛化到 N 人 + "急停/亮灯"变体。发→亮灯→抢拍→系统判 |
| 3 | **题库/quiz `prop.quiz`** | B/C | 卡 1(科目一)+ 118 原子(问答题目全类)+ 60 关键词命中 | 中:局长现可自出自判,缺的是题目权威与答案校验;量大 | 中:接题库资产(如驾考)+ 出题到玩家端 + 客观答案自动判。发→答→系统校验→回执 |
| 4 | **秒表/竞速计时 `prop.stopwatch`** | A/B | 卡 1(开黑登录赛)+ ~15–20 原子(最快/最慢/竞速) | 中:`timer` 只做时长,缺精确"排名/谁最快"。与 #2 部分重叠 | 中:各端计时 + 系统排名。发→各自计时→后台排名→公开 |
| 5 | **计数器/累计器 `state.counter`** | A | 0 卡 + ~10 原子(数字炸弹范围/共杯累计/炸弹接龙计数) | 低-中:多数可 `state.note` 硬扛,专用计数更干净 | 小:state 子操作,记数/范围/阈值触发。无玩家端动作,纯账本 |
| 6 | **虚拟转盘 `prop.wheel`** | A | 卡 2(转瓶/指向,已被 `random.pick` 覆盖)+ 28 原子(多为随机点人) | 极低:`random.pick` 已闭环,仅仪式感 polish | 小:random.pick 加转盘动画皮,不改机制 |
| 7 | **虚拟硬币 `prop.coin`** | A | 卡 0 + 3 原子(且 1 条是物理拍桌翻币) | 近零:全库几乎没有抛硬币玩法 | 小:但无需求,不建议单独做 |

**结论**:直觉里的"转盘/虚拟硬币/虚拟翻牌"三件套,只有牌(且是**牌桌级**,不止翻单张)真正值钱;硬币、转盘是伪缺口。真正的两大缺口是 **虚拟牌桌** 与 **抢按/反应仪**,分别吃掉 E 型的两个硬断链主簇(牌桌 25 条、闭眼 6 条 + 反应退化一大片)。

## 六、【道具箱】主持词增补建议(条目化)

只诊断不动刀,以下为将来改 prompt 时的候选条目。原则:在现有五型里能接住的家族,prompt 里点名走哪型;接不住的(E)先标注"暂缺原语,降级处理",别让局长被逼虚构。

现面能接住、建议 prompt 点名的:

1. **骰家族 → A**(已落):大话骰/私骰走 `prop.dice_cup`,叫价在桌上嘴对嘴、局长只当开牌判官。**保持,勿回退到 random.dice 替玩。**
2. **随机点人/转瓶/指向 → A**:凡"随机指一个人/转瓶决定",一律 `random.pick`(带 visibility 按需),**禁止局长嘴上"我来指一个"挑受害者**——那既是虚构随机、又是局长挑人。可加一句:"转瓶/指向/随意点人=random.pick,不许口头点。"
3. **隐藏信息(藏物猜手/猜单双/毒杯号)→ A 私密**:`random.pick/int 自己看 + player`,答案走下一拍回执,揭晓再宣布。已在 prompt,建议对"藏物/猜数"家族再点名。
4. **线下物理输家(抓手指/传递/竞速/条件查验)→ C**:输家桌上可见,`ask` 问一嘴或等 `say` 上报,`state.note` 入账。建议明确:**这类高频循环款,每轮一次 ask("这轮谁被抓/谁溢出?")即可,别每轮长篇复盘**——契合【循环开】。
5. **达标主观(跳舞/模仿/彩虹屁/真心话)→ D**:`ask` + options 让全桌多数认,或 `judge.photo/audio`(造型/声音类)。建议点名:"'跳得好不好/像不像'不是你说了算,开 ask 让桌上投,或 judge 拍照/录音判。"
6. **敢不敢/挑战完成 → C(信号)**:结算看 done/forfeit 事件,别替玩家判"他敢了"。已在【玩家三信号】,建议在道具箱回指一句。

现面接不住、建议标"暂缺原语·降级"的(E,防被逼虚构):

7. **闭眼先睁眼型 → 暂缺(抢按仪未落)**:prompt 应明说"全员闭眼类,系统看不见谁先睁眼,别假装判——改用快枪手 `duel` 那类系统亮灯抢拍,或直接换一个游戏"。**这是大话骰同病的下一个坑,优先在 prompt 里堵。**
8. **有状态牌桌(斗地主/田忌/骗子酒馆/变色龙等)→ 暂缺(牌库未落)**:prompt 应标"多轮手牌类现无牌库原语,别口报发牌/记牌(那是虚构+替玩),抽到就 `state.discard` 留痕换一个,直到 `prop.deck` 上线"。
9. **接龙丢字/复述 → C 降级 + 轮次器**:用 `ask mode=轮流` 逐人开窗,局长把累积句记 `state.note` 当裁判据;明说"判丢字靠你听 say + 记的句子,听不清(inaudible)不硬判,超时才罚"。
10. **最快/最慢竞速(登录赛/松手最快)→ C 降级**:明说"无秒表原语,别编毫秒名次;改问'谁明显最慢'桌上认,或换成 duel 二人对决定胜负"。

落地次序建议(与第五节排行一致):先补 **虚拟牌桌**(解锁最大、纯断链)与 **抢按/反应仪**(闭眼硬断链 + 反应簇升型),题库/秒表次之,转盘/硬币不做。
