## 0. 适用范围与模式分叉

双模各按各自的尺量，**不互相迁就**（决议纪要 §31/§32）：

| 通道 | 收货标准 | 本单对应节 |
|---|---|---|
| 普通模式（Mode A，预编译便宜档） | **四绿链** | §C |
| 狂野模式（Mode B，活局长浓局档） | **真人局四问**（四个二值+一句话） | §D |
| UGC / 狂野农场入库 | **存活桥三条件**（离线·单向·过闸=走四绿链） | §E |
| 生产线切换 | **切换门**（双绿才切） | §F |

> 分家红利：狂野生金、普通复刻——但**真桌出身不免检**，回流普通必走 §E（决议纪要 §32/§34）。

## 2. 普通模式验收 · 四绿链

首件四绿 = 里程碑（决议纪要 §29：三绿件 haiku_A_v21r2_01_r1 真桌通过 → 三绿升四绿，golden v2.1 首金转正）。

### 绿① 结构闸 — check.py v2.1
- [ ] 全量过 check.py v2.1；**expr 硬闸**：`source:expr` 须为可解析表达式且引用 state 键，散文自由文本拒（活证 dsT_A_v20_01）。
- [ ] **dead_prop 读表豁免**：只查"可引用类"，免引用件（沙漏/记分板/匿名投票器/公共看板）按正典表豁免，check 读表不硬编码。
- [ ] 软闸落旁车 `<件名>.warnings.json`，**被检件保持纯设计层**（发令包 线二 / 范围 §3）。

### 绿② 判级 — 裁判 spec-judge-v0.3（落籍正典，sha df36…6ec25）
- [ ] 全量出 `.review3.md`，判"过"。
- [ ] **SOP 抽审**：首扫全套 4 份 = 2 破 + 2 过（过件不足则全部过件 + 破补足至 4）；任一变动若靠含混通道硬凑即打回（评级尺 §4 / 派发包【裁判】）。

### 绿③ 编译 — 至 finale
- [ ] 过闸且判过件喂编译器，**跑到 finale**（三章缺一不认，派发包 阈值⑤）。
- [ ] 编译器 **7 模板全量零回归**（范围 §8 / 排期切换门）。

### 绿④ 真桌 — 人局通过
- [ ] 真人局完局，**三个二值全绿**：完局 ✓ / 零卡壳 ✓ / 还想再来 ✓（时长与笑点按 §28 校准不追，决议纪要 §29）。

> **批级阈值（验证批场景附加，五条终值·跑后不改，派发包 §阈值拍板）**：
> ① 语义包疗效 = Sonnet 判级 pass@1（基线 12.5%）+ "口头/意图/状态无通道"族破因计数（族计数减半为有效）；
> ② 教学疗效 = params 首过率对基线 62.5/25/25（Sonnet ≥90% = 解释义务解除线）；
> ③ 看点六 = 覆盖数 / 前四占比（对 78.7%）/ 真 dead_prop 率，记录性；
> ④ 切换门（见 §F）；⑤ 三绿件（本节绿①②③）。

## 3. 狂野模式验收 · 真人局四问

不上编译、不上判级——**桌子说了算**（决议纪要 §30/§31）。

- [ ] 二值一：完局？
- [ ] 二值二：零卡壳？
- [ ] 二值三：还想再来？
- [ ] 二值四：（对照题）四绿件基线 vs 活局长一晚，哪桌更好？
- [ ] 一句话现场判语（记档，不评分）。

> 三条底线内百无禁忌：档位约定 / 拒绝权 / 房主最大（决议纪要 §33 活局长 v0）。
> 兜底=局长口才，非系统功能；风险不设门（$0.3–0.6/局、1–3s 回合延迟、状态漂移照记不拦，§30）。

## 4. UGC / 狂野农场入库 · 存活桥三条件

狂野好局 → 普通货架，合法性三条件缺一不可（决议纪要 §32，用户共识点）：

- [ ] **离线**：局散后整理，非局中（局中两楼不相往来）。
- [ ] **单向**：狂野 → 普通（传送带单向开动）。
- [ ] **过闸**：照走 §C 四绿链，真桌出身**不免检**。

> 收编三源一链：官方采集 / 狂野农场 / 创作者 DIY 上传 → 四绿链审核上架（决议纪要 §35）。

## 5. 切换门 · 生产线切换（双绿才切）

- [ ] Sonnet 判级 **≥30% 档**（≈3 发/合格局）。
- [ ] 编译器 **7 模板全量零回归**。
