# 门槛验收单 v1 草案(2026-07-18 房主交正例七则,Fable 翻译;待照准)

> 依据:`门槛验收单-征集说明与模板.md` 流程。房主关键原话三句:
> ①综艺壳"也不是不行,就是感觉没有灵气"(→ 首条否决条改判);
> ②"我不想举例限制 AI 的发挥,我要的是无穷可能,我给的只是我的一个面,
> 每个人的面都不一样";③"核心就是惩罚要够搞"。

## 一、房主正例存证(七则,摘记)

1. 兴趣=JoJo,场景扫描到发胶 → 惩罚:发胶梳东方仗助头,AI 视觉验收。
2. 兴趣=海贼王 → 三个酒瓶当刀摆索隆三刀流造型+说台词。
3. 兴趣=小黄人,感知到酒吧人多 → 起立大声模仿小黄人台词;AI 供参考图/音,调分贝计验收。
4. 兴趣=最近痴迷的电影 → 联网搜名场面,AI 给站位辅助,两人复刻男女主场面。
5. 兴趣=C罗 → 模仿 "Siuuu" 名梗+动作。
6. 兴趣=周杰伦 → 抢答歌词最快者赢、输的喝;或模仿周氏语调,AI 打分谁更像。
7. 兴趣=无畏契约 → AI 检索该游戏当下热度最高主播(康康)→ 模仿其经典梗。

## 二、灵气五性质(从七例提炼,可判定)

1. **痴迷命中**:内容从**这一桌这个人**上传的兴趣里长出来,且打到该兴趣内部
   名场面/名梗粒度(是"仗助的发型",不是"摆个 JoJo 姿势")。
2. **现场嫁接**:AI 感知到的现场实物/状态被**错位挪用**进挑战(酒瓶=刀、
   发胶=造型工具、酒吧人多=公开处刑舞台)。
3. **场面产出**:惩罚产出**可目击可拍摄的名场面**,不是分数变动。"够搞"的
   可判定translation = 完成瞬间全场爆笑 + 有人想掏手机拍。
4. **当下热度**:文化梗优先取当下最热且有检索依据(康康案),不吃老本。
5. **AI 全链**:AI 既当**导演**(供参考图/音/片段/站位)又当**裁判**
   (视觉/听觉/分贝验收)——不是念题机。

## 三、验收条目(线待照准时定;建议:硬条全过 + 软条 ≥2 = 过门槛件)

**否决条**
- N1(修订版):**零耦合即毙**——环节/惩罚可整段搬到任何一桌任何人而无任何
  损失(与本桌玩家画像、现场实物、当下文化三者全部零耦合)= 无灵气,毙。
  ~~综艺四段壳即毙~~(房主 2026-07-18 改判:壳本身不毙,没灵气才毙)。

**硬条**
- H1 痴迷耦合:每局 ≥N 个环节由玩家兴趣画像直接生成,其中至少一半达到
  "兴趣内部名场面/名梗"粒度。
- H2 现场嫁接:每局 ≥N 次调用感知到的现场实物/状态,且为错位挪用(非本用途)。
- H3 场面产出:惩罚主体必须是表演性动作(造型/喊/演/唱/模仿);纯扣分、
  纯口头答题不得作为惩罚主体。
- H4 AI 双角色:挑战须含「AI 参考素材供给」+「AI 感知验收」两环。

**软条(计分)**
- S1 时效热梗(检索依据的当下梗)。
- S2 复刻类给站位/教学辅助。
- S3 结算极简:输的喝/一句话结算——**复杂度预算全花在内容上,不花在规则上**
  (七例结算全为 R0/R1,此为房主审美的结构侧特征)。
- S4 传播钩子:产出画面天然适合拍段子(AI 视频推广的素材自产)。

## 四、例子≠白名单(房主原则成文,验收铁则)

七例定义的是**生成函数**:f(玩家痴迷 × 现场实物 × 当下热梗)→ 表演性惩罚/挑战。
**验收对"性质"不对"题材"**——枚举永不封顶,出现七例之外的新题材新形态,只要
五性质在,就是好件;复刻七例本身反而应视为偷懒信号。每个玩家的面不同,
系统的本职是挖出每桌自己的面。

## 五·前置:房主二轮补充(同日,v1.1 追加)

**正例补录(三型新物种):**
8. 怪奇物语 → 模仿小十一施念力;**或授予技能**:本局对准一人做施法动作+念台词,
   该人受罚/喝——梗变成**可持有的权力**。
9. 哈利波特 → 场景扫描到可当魔杖之物 + 有玩家画像含 HP → 授技能:该物本局
   即你的魔杖,指人念"啊哇肯大瓜"该人喝,**限用十次**。
10. 反向案(房主问"会出现吗"→ 答:会,函数改双向):扫描到不用的丝袜 →
    丝袜套头模仿电影劫匪——**从实物出发反查文化库**,不必挂在任何人的画像上。
11. 调制案(房主问"会出现吗"→ 答:会,立调制器层):二十秒连续 Siu 十下
    摆姿势——同一挑战 × 机制参数 = 不同难度不同玩法。

**由此新立三算子/概念(生成函数 v1.1):**
- **双向嫁接**:痴迷→找实物(正向)与 实物→查文化库(反向)皆合法;
  反向案天然全桌共享,不吃画像。
- **难度调制器**:计时/次数/连击/同时/接龙/加注等参数层,把一个原子调制成
  难度族——复用既有 params 思想(limit/seconds),原子库按"基础挑战+调制器"
  两层存,一条原子生成一梯子。
- **惩罚三向**:①结果型(输的喝);②**权力型(技能卡)**——惩罚权作为
  可持有道具(绑实物、限次数、限本局),持有者主动点名,创造轮内张力;
  ③事件型(触发条件全场)。8/9 例即权力型,原子 schema 的 atom_type
  增「技能授予」,带 uses/duration/trigger_action/effect 字段。

**过程>结果(房主定调,升格为总原则):**
- 原话要旨:喝酒局里加减分本身没意义;没人管最终赢家,**享受玩的每时每刻
  才是全部**。社交场景重过程,竞技场景才重结果。
- 落到系统:①账本重新定位=**过程发动机**(驱动谁喝/谁演/谁被点名),不是
  胜负记录仪;②终局仪式保留但轻量(prompt v0 的冠军+称号+合影梗,是仪式
  不是竞技);③**过程质量已有现成量尺——笑声账本口径**(first_laugh/笑声
  密度/dead_air):episode 复盘按时刻密度评,不按胜负悬念评;④惩罚货币
  不硬编码为酒:喝/表演/怪造型同为结算货币(替代结算开关兼容学生局)。
- 房主自省"例子全是喝"照实记:frame 而非裁定,系统不得据此把酒写死。

## 五、对系统的直接技术推论(回写各设计案)

1. **开局配置补「兴趣画像」**:每人上传几个痴迷(动漫/球星/歌手/游戏/电影)
   ——input 的「素材」字段升维,两模通用。
2. **素材包预拉取**:开局阶段按画像联网检索热梗+参考图/音,备好本桌弹药包;
   把检索延迟移出回合循环。工具面第九件候选 `hunt`(检索):开局预拉为主、
   现场按需为辅。
3. **perceive 优先级上调**:七例中五例依赖感知(场景扫描/视觉验收/听觉验收/
   分贝计)——perceive 从"候选"转"必建",排 M2 后立即施工。
4. **结算极简倒置**:房主审美 = 结构极简 + 内容炸裂,与 Mode A 现状
  (结构复杂 + 内容综艺)恰好倒置——skill v2.2 反同质化草案照此对表修订。
5. 版权注:参考素材走检索/外链展示,不内置分发影视片段;细则属运营期,
   按 2026-07-18 宽尺度裁定不预设过度限制。
