# settlement 表达力债 · 登记（v3 候选，预注册触发器把门）

> 缘起：用户点题（2026-07-13）——阵营胜利/联盟/内奸/平局语义/纯玩无胜负，schema 未覆盖。
> 定性：**登记债，非漏洞**——现行闸（check.py win 有源 + 裁判 7c/e + 三键必填）对边界外内容
> 一律拒收、不误放。方向安全，代价是表达力。
> 纪律：分拣判据——玩法表达力 = "游戏该有什么" → 等验证数据；键集冻结，触发前不动 schema。

## 1. 当前表达力（v2.0 事实）

个人计分制：scoring(±N/惩罚三档) + win(只引用 scoring 分量) + tiebreak(分数轴平局)。
身份类内容的合法形态 = **终场一猜支线**（裁判第 10 项例外通道 + `identity_resolution:
terminal|reshuffle`），身份价值折算进个人分——已铺好，生产可用。

## 2. 边界外四类与现行处置

| 类 | 缺什么 | 现行处置 |
|---|---|---|
| 阵营胜利（狼胜/民胜） | win 判据引用 state（存活/身份）而非分量 | 折算走终场一猜支线；多轮放逐社推 N≤5 被硬否决，N≥6 理论有客户、**迄今零产物证据** |
| 联盟/内奸多方结算 | win 多互斥分支 + 身份组合判定 | 无表达；写进来被闸拒收 |
| 阵营平局/双败/共胜 | tiebreak 只在分数轴 | 随上两类走 |
| **R1 单局惩罚循环·无终局**（Opus R0–R3 轴补录） | 终局条件本身不存在：每小局产出中招者，惩罚当场兑现当场清零 | **最大逼形嫌疑**——现行 skill 会把击鼓传花类强行包成"N 轮计分"的 R3；裸 R1 写进来被三键必填拒收 |
| 纯玩无胜负（仪式类，Opus 坐标=R0） | mode 判别器 | **故意判死**（三键必填是冲结算失效类立的闸）；归宿是氛围工具产品面，不走"局"生产线 |

**Opus 坐标下的现 schema 表达力一句话**：R3 全支持；R2 半支持（淘汰可经 on_named.eliminate，win 仍须引 scoring 分量）；R1/R0 逼形或拒收。

## 3. v3 候选清单（触发后才进设计）

1. `win` 结构化：多互斥分支 + 判据可引用 state 键（阵营/存活）——覆盖阵营胜与内奸。
2. 阵营结算原子：scoring.who 增阵营选择器 + 阵营平局语义。
3. **settlement 结构判别器（R0–R3 枚举）**——取代原"mode: scored|ritual"布尔案：R3=现 schema，
   R2=补淘汰终局语义，R1=惩罚循环无终局，R0=仪式。仅触发器亮灯才设计；R0 若归氛围工具面则永不做。
