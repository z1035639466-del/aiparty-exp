## 3. 预注册看点（判读指标 + 触发器；⚠ 标注处为建议阈值，跑前需拍板）

**看点一：settlement 是否焊住了结算类破。**
指标：结算/终局/平局类破因计数与占比，分模型，对 v1.8 基线（基线事实：Sonnet 的破全是
lint 形状——数值未定义/结算对象矛盾/弱占优）。
预期：Sonnet 结算类破趋零 → pass@1 上探。
⚠ 触发器：若 Sonnet 破因中结算类仍占 ≥1/3 → 判"字段没焊住"，查泄漏点在 prompt 措辞还是
check.py 漏检，修完再跑，不带病出路由表。

**看点二：弱模型是否仍往候选池型撞（阶段 3 菜单过滤的砝码）。**
指标：第 10 项硬否决触发数 + shape 标记中候选池型核心循环数，分模型。
预期：Sonnet ≈0（文本禁令对它够用）。
⚠ 触发器：Haiku 或 DS 任一 ≥2 次/8 发 → "prompt 自检效力随能力衰减"再添实锤，
阶段 3 的"按 N 过滤词汇菜单"（v1.9 第 7 项移交件）优先级升为编译站首件。

**看点三：B×20 shape 分布（套皮问题终审）。**
指标：shape 去重分布 + 有效机制家族覆盖（用膨胀字段合并后计数）。
⚠ 触发器（梯子制度，亮了才动、不跳级）：单一 shape ≥60%，或前二机制家族合计覆盖 ≥70%
→ 判"套皮未解"，升级到内容层弹药包（扩库仍不动）。
未触发 → 套皮案以"prompt 修复有效"结案。
