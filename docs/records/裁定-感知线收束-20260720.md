# 裁定纪要 · 感知线收束(2026-07-20,房主裁定,Fable 落地)

## 裁定

1. **笑声信号、全程录音、能量窗口、分贝监听:全部不做,从活文本中除名。**
   理由(房主):真机没有这个信号;就算全程采集,也分不清是玩游戏笑还是
   聊天笑,采回来全是脏信号,给 AI 也调试不明白;好玩不好玩的品味判断
   由房主复盘 episode 直接迭代——"跑出来看结果迭代就完事了"。
2. **按钮档 = 真机基线**,不是降级档。主持在真机上的全部感官:
   三信号按键 + 定向发言(to=局长)+ 计时器。
3. **转写档 = 开发台上帝视角仪器**(agent 桌的 say 本来就是文字),
   测理想上限用,非产品形态。
4. 唯一保留的感知形态:**主持显式发起的判定时刻**(拍照验收类),
   属多模态判定通道,合并后档,与常驻监听无关。
