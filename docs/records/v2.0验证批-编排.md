## 4. 预注册看点（阈值已拍板 2026-07-13 · 终值，跑后不改）

**一：settlement+scoring_ref 是否焊住结算类破。** 指标=结算/终局/平局类破因计数占比，分模型，
对 v1.8 基线（基线事实：Sonnet 的破全是 lint 形状）。触发器：Sonnet 破因中结算类占比 ≥1/3
→ 判"没焊住"，查泄漏在 prompt 措辞 / check.py 漏检，修完再跑，不带病出路由表。
**分母下限：Sonnet 破因总计 ≥6 方可判定**；不足则并入重试批判词合并计数后复核，仍不足记
"样本不足·不判"——小样本占比既不得触发、也不得据以结案。

**二：弱模型是否仍往候选池型撞。**（**件头注明：排期信号，非缺陷率**——量的是文本禁令对弱模型
的效力衰减，用途是给阶段 3 菜单过滤排期，不计入路由表的模型缺陷统计；报告件头照抄此注。）
指标=第 10 项硬否决触发数 + shape 候选池型核心循环数，分模型。
触发器：Haiku 或 DS 任一 ≥2 次/8 发 → 阶段 3"按 N 过滤词汇菜单"升为编译站首件。

**三：B×20 shape 分布（套皮终审）。** 指标=shape 去重分布 + 有效机制家族覆盖（膨胀字段合并后）。
**v1.8 基线（ds_B_div_shapes.md，2026-07-13 落档）：社推 55%（11/20）、前二家族约 50%、
覆盖约 6 族**——基线只作对照解读，触发器仍按下列绝对值对 v2.0 批判。出表前两侧家族名
统一映射到 36 家族正典表（基线为自由文本临时聚类，须归一）。
触发器（三条任一）：单一 shape ≥40%；前二家族合计 ≥50%；**家族覆盖 <6 个**
→ 判"套皮未解"，梯子升内容层弹药包（扩库不动）。
未触发 → 记 **"本轮未复现"**——案不销，本组触发器随后续跑批常挂，不作"prompt 修复有效"的终局结论。
候选根因登记（2026-07-13，用户）：道具原语过少 → 与"素材导向""词汇菜单未按 N 过滤"同列，
由 B×20 终件 shape 表裁；即便坐实，处置按梯子（弹药包先于扩库）。

**四（记录性，不设触发器，首批立基线）：params 结构合规率分模型。** 指标=check.py v2.0
首过率 + 软闸 warning 密度（prose_param_mismatch / dead_ledger / dead_prop 每件均数），分模型。
弱模型写结构化参数与写散文的能力差是路由表的新轴，本批白捡。
**强制解释义务：Sonnet 首过率 <90% → 出表必须附根因归析**（prompt 措辞 / schema 设计 /
模型能力，三分或组合）——只解释、不自动动作，不构成触发器。顺手预留列：**可编译率**
（MVP 7 覆盖内的过件喂引擎编译器的通过率）——编译器 §4③ 完工前留空。

**五（⚠ 待遇待拍——建议同看点四档）：道具僭越率分模型。** 指标=让道具承担判定的件比例
（机器槽渗漏由 C7 硬闸拒件、计入结构 fail；散文层僭越由裁判第 8 项子类判破、判词点名），
分模型——测模型有没有把物理世界当裁判用，是 C7 的直接落地量。
⚠ 建议待遇：记录性、不设触发器、首批立基线（零先验，凭空拍数会被"跑后不改"锁死）；
**Sonnet 僭越率 >10% → 强制解释义务**（n=8 下即 ≥1 件；盯强模型的逻辑同看点一/四——
强模型都拿柠檬当裁判，是正典没讲清或 schema 引诱，不是模型不行。方向与看点四相反：
僭越越低越好）。
