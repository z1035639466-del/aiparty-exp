# 道具原语 · 实物绑定层设计 v0(2026-07-18 房主立意,Fable 落档)

> 房主原话要旨:把道具原语和现场某个物品挂钩,并给出对应使用条件(要好玩、
> 增加戏剧性);现场没实物,DM 直接调用原语;魔杖本身就是一条道具原语,
> 现场物品只是匹配上了它。**另一个关键:道具原语要够多。**
> 正典承接:《道具库 v0 定稿》(docs/specs/spec-prop-library-v0-final.md)——
> 其 1.1 权限铁律、1.3 可见性引擎、1.4 现实物品地基全部沿用,本案是其扩建层。

## 一、术语两层分离(防混用,E 档表述随之校准)

- **道具原语(语义层)**:游戏效果模板——魔杖/时间暂停器/测谎仪。正典道具库
  的对象,「真人可写、AI 只读只调」白名单铁律照旧(1.1:这是安全模型前提)。
- **道具组件(实现层)**:渲染与交互代码——卡组渲染/计时器/分贝计。E0–E5
  工作量尺和「按解锁原子数排序」的 roadmap 说的是这一层。
- 关系:**原语调用组件**。"时间暂停器"(原语)调用 timer+perceive(组件);
  组件够用时新原语=纯配置(E1),这就是原语能快速扩产的结构原因。

## 二、道具原语的双态存在

| 态 | 形态 | 例 |
|---|---|---|
| **实物绑定态** | 开局扫描→匹配→授予:"这根筷子今晚是魔杖" | 魔杖绑棒状物;时间暂停器绑遥控器/打火机 |
| **虚拟态** | 现场无合适实物,DM(活局长)直接调用原语,系统仪式补皮 | 全屏红光+音效即"时间暂停"发动 |

**绑定只换皮,不改机制**——机制永远封在原语里,AI 只做匹配与命名,不碰效果
定义。故白名单安全闸不破:绑定层再花,合法清单仍是那一张。

## 三、绑定流程(开局配局阶段)

1. perceive 场景扫描 → 现场物品清单;
2. 匹配打分:形态相似度 × 文化梗加权 × **玩家画像加权**(HP 迷在场→棒状物
   优先绑魔杖;怪奇物语迷在场→念力向原语升权);
3. 授予设计:给谁(点名/抽签/成就解锁)、何时激活(开局即有/气氛峰值解锁)、
   **使用条件仪式**——动作+台词必须做全才生效(仪式做不全=发动失败本身就是
   笑点),条件设计以验收单「够搞」判定(全场爆笑+有人掏手机);
4. 次数/时效/野度/S 位随原语定义走,账本经 state 记录用量。

## 四、原语扩产工单(「要够多」的执行形态)

- 现状 14 件(工具箱@2026-07)。**目标:v1 扩编批 50+。**
- 生产矩阵:**文化效果梗**(时停/魔杖/测谎/读心/护盾/隐身/交换/复读/连坐/
  放大镜/免死金牌/巴掌券…)× **效果类型**(冻结/强制/转移/侦测/豁免/复制/
  加倍/连坐/反弹)× **使用条件模板**(限次/限时机/解锁条件/对象限制)。
- **生产纪律(正典 1.1 铁律的扩产版)**:子 agent 只批产**候选草案**(每条含
  效果定义/使用条件/判定通道/野度/S位/可绑实物类型/所需组件),**入库必经
  房主照准**——AI 起草,真人盖章,库再大白名单性质不变。
- 另一供给源:采集原子中「技能授予/道具挑战」型反推原语候选(整合线聚类产出)。

## 五、正典修订候选(待照准,不擅改 spec)

道具库 v0 §1.2 筛选准则三短板(隐藏/随机/记忆·裁判)建议增第四类:
**戏剧引擎**——现实做不到的"超能力仪式框架"(时停/魔法/审判)。时间暂停器
判"谁动了"仍吃记忆·裁判+perceive,但其入库理由是戏剧性而非补短板,现行
三条收不进它,故须扩准则。此为 spec-prop-library 的修订案,房主点头后进正典。
