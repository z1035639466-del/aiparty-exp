## 病类定义(本次四查)

- **A 角色归并**:原文是不对称角色(一人当鬼/庄/陷阱,其余人另一种动作),skeleton 摊平成「众人各自同时做全部动作」。
- **B 肢体预算超支**:skeleton 要求单人同时的动作超过两手两脚一嘴(比心占两手、持杯占一手、闭眼影响判定等,按常识彩排)。
- **C 变体假互证**:variants 原文近乎逐字相同(互抄模因),歧义处标注「证据独立性≈1」,提示该卡歧义未被消解。
- **D 顺带硬伤**:方向反了/人数错了/道具漏了/结算对象错了等明显对不上;**语义对不算病,不扩大到风格与措辞**。

判级口径(本报告内部一致性):
- **必修**:高置信、原文清晰且与 skeleton 硬冲突,照 skeleton 跑出来是另一个游戏(物理不可能、抢/陷阱颠倒、失败判据写错、变体污染)。
- **可疑**:读法偏移可能改变玩法,但原文 OCR 残破/变体记忆化/属解释性 gap-fill,证据不足以拍死;或改动虽真但仅动到细节。
- **健康**:skeleton 忠实还原原文语义(留白合理、gap-fill 合理均算健康)。

数据观察:变体记忆化(逐字互抄)在本库极其普遍(如 v1-01 七条、v1-03 六条几乎逐字重复),**但只有在留下未消解歧义时才计为 C 病**;机制清晰的记忆化重复不计病。另跨卡机制重复(三张传冰 v1-19/35/40、两张共杯轮倒 v1-31/39)是去重议题,不属四病,仅附注。

### pat-v1-08 叠指夺杯 —— A/D:把「持杯陷阱」误蒸成「抢杯竞速」

**原文摘录**(xhs-00595,清晰非残破):
> 做一个空中举杯的动作,所有人学完后,拿两杯酒放一个人的手上,然后伸出大拇指叠罗汉,拿杯的人没有手,所以只能喝酒

(xhs-00707/01626/01651/02714 同义,均为「两杯扣在**某一人**手上→其余人叠拇指→持杯者无手可玩→喝」。)

**现 skeleton:**
> 领做者示范举杯手势 → 号令时众人改叠拇指抢杯 → 抢到/未抢到者入结算

**病灶:** 原文是**不对称陷阱**——两只杯子预先扣在**指定一人**双手上,占死其双手;其余人伸拇指叠罗汉,持杯者根本无手参与,「必然」垫底喝酒。这是与 pat-v1-09「持杯必输」同构的整蛊(只是把抓手指换成拇指叠罗汉)。skeleton 把它读成了「众人抢一只中央的杯子」的竞速赛(A 角色归并:预设受害者被抹掉;D:凭空造出「抢杯」动作与「抢到/未抢到」判据,原文无「抢」)。demo_prompt 第二、三格更把错读固化(「桌面中央放两个空杯…众人的手悬在杯子上方…一只手已抢先按在杯子上」),会照错画。

**建议改法(skeleton):**
> 领做者示范空中举杯、众人跟做 → 把两只杯子扣在其中一人双手上占死其手 → 其余人伸拇指叠罗汉,持杯者无手可玩必然垫底 → 持杯者入结算

**建议改法(demo_prompt):** 三格,只画动作与队形,禁酒液/品牌/文字——第一格:围坐的手臂,一只手做空手举杯,其余人跟着模仿;第二格:两个杯子轮廓分别扣放在**同一个人的左右手掌上**,该人双手被占;第三格:其余人伸出拇指叠成塔,持杯者双手端着杯子无法加入,被单独框出。

## 物理彩排关卡建议(将来做成 tools/ 常驻质检脚本的判据草案)

面向蒸馏管线:在「原子簇 → pattern skeleton/demo」这步之后、入库之前,挂一道自动+半自动的「物理彩排」关卡。以下为判据条目化草案(只列判据,不写代码):

**1. 肢体预算表(治 B 超支)**
- 为每个 skeleton 抽取「同一角色需同时保持的动作」清单,按预算核对:手≤2、脚≤2、嘴≤1、眼(闭眼即判定失明)。
- 硬冲突词典:比心/双手比划=占 2 手;各持一杯=占 1 手,两杯=占 2 手;闭眼=该角色本回合不可承担「看/传/瞄准」类任务。
- 触发:某角色手数>2 或「占手动作+需手动作」并存 → 打回。爱心抓手指即此关应拦下。

**2. 角色对称性核对(治 A 归并)**
- 从原文抽「主语切换」信号:出现「一人/其中一个/庄家/主持…其余/大家/每个人…」的对举句式,判定为**不对称角色卡**,强制 skeleton 保留至少两类角色(陷阱/庄 vs 众人)。
- 反向护栏:skeleton 用「众人各自同时…」这类全称量词覆盖了原文里存在的单人角色 → 打回。
- 与卡名一致性:卡名含「围圈/中心/居中/一人」等结构词时,skeleton 队形须与卡名不矛盾(治 t1-02 型自相矛盾)。

**3. 结算对象/判据回指(治 D 谁喝错)**
- 逐卡抽出原文的失败判据关键词(丢字/拉断/最后一个/落单/没有的/先睁眼…)与 skeleton 的结算判据做词面比对,不一致即高亮人工复核。
- 特别名单:近义误替(丢字↔用字重复、拉断↔松开、主界面↔对局、无任何 X↔缺指定 X),命中即打回。

**4. 变体独立性评分(治 C 假互证)**
- 对同一卡的 variants 两两算文本相似度;若簇内近逐字重复(相似度阈值以上),该卡「证据独立性」记≈1,并强制:凡 skeleton 有留白/歧义处,必须至少一条**独立表述**变体支撑,否则标「未消解歧义」。
- **推广/模板尾巴剥离**:入相似度前先剔除已知营销模板句(如「00种玩法…评论区大胆说出你的小心思」)。**只用可执行规则段做聚类与互证**,防止靠共享垃圾文本假聚类(pat-v1-53 正是此漏)。
- 簇内规则段互斥检测:剥离尾巴后若变体的动作/道具/结算彼此矛盾(v1-53 的比大小 vs 伸指定格 vs 闭眼),判为「污染簇」,拆卡。

**5. 原文覆盖率与缺原子闸(治宁空毋编)**
- variants 里引用的 atom_id 必须都能在原子库取到 text_raw;出现 `open-xx` 等库外 id 即标「原文缺失」,该卡不得据缺失变体消解歧义(t1-02 的 open-04、t1-05 的 open-02)。
- skeleton 每个非留白断言都应能回指到某条变体原文的字面;找不到回指的断言标「疑似先验重建」,需真人彩排确认(pat-v1-44 型)。

**6. 一次真人/心智彩排闸(总兜底)**
- 上述 1–5 通过后,仍强制一句话「心智彩排」:让评审按 skeleton 逐字扮演一遍,数手、数角色、走一遍结算;跑不通即回炉。这是背景四层根因里唯一缺失的「没人数手」关卡,应作为入库前最后一道硬闸。
