# 裁定纪要 · 六桌实测三条(2026-07-20,Fable 裁,依据 M2-藏信息六桌实测报告)

## 一、私发半盲 → 荷官回执(形态二的引擎版,当日已实装)

**裁定:遮蔽按观看者定,不按出口定。** 原表述"出服务端的一律是遮蔽版"修正为:
每个主体拿到且只拿到自己有权看的面——玩家=`/api/view`,房主=驾驶舱全知,
**发牌人=自己发过的牌**。荷官看不见自己刚发的毒杯号,公平随机就退化成手工指定
(4 人桌实锤),这不是保密,是自残。

- 形态一(回合响应给原文)**否**:`/api/turn` 响应在开发台是共享页面,给了等于全漏。
- 形态三(写说明书认栽)**否**:等于承认公平随机在隐藏身份上不可用,把功能做没了。
- **形态二成立,且真机路径有更深的坑**:`LLMDriver` 的上行从来只有 digest+events,
  **主持在真机路径上看不到任何工具结果**——私发半盲、"嘴上给了撒谎权账本没有"、
  "不信 tally 自己读原始事件"三个症状同根。

**实装(engine)**:`tool_receipts` 事件——主持上一拍工具的真实结果(私发原文/
私密摇点数/钳制记录)只走 driver 专用信道注入下一拍上行,**不进 line、不进
events_in、不进任何 HTTP 面**;prompt 加【荷官回执】(仅你可见/不许念/钳制必圆场)。
**待办(归 Opus,模拟台)**:harness 里 HTTP 挂载的主持走不到 driver 信道,
加 `GET /api/dealt`(只回主持自己发过的私件与私密摇结果)供其查回执;重跑时
主持 agent 改读此口。

## 二、主持中立性 → 写进 prompt(当日已实装)

**裁定:入铁律层级。** 实测一句"滑得跟泥鳅一样"= 六票全跟,主持在推理局里
的即兴吐槽就是权重最大的一票。落法【推理局中立】:投票/嫌疑窗口开着时只主持
流程不下判断;吐槽留到结算后;**嘴瓢带了节奏当拍认账自罚**——"这杯我陪一口"
被三名玩家独立列为全场亮点,认账姿势本身是内容,不禁。

## 三、主持静默唤醒 → 冷场闹钟(当日已实装)

**裁定:等待权补上"发现没人了"的另一半,唤醒≠催促。** 主持行动后场上静默
90 秒(`SILENCE_WAKE_S`),注入一次 `table_silent{quiet_s}` 叫醒;一段静默只叫
一次,任何桌面动静解除闹钟,空拍(静等)不重挂。催不催仍归铁律(同一件事
≤1 次);prompt 加【冷场】:分不清"专心做挑战"还是"人走了",问一嘴合法,
催过就换环节/降档/收局。2 人桌冻死在第 7 拍的形态就此终结。
