## 二、四条关键结论(Fable 裁:全部采纳入设计公理层)

1. **聋主持自发收敛到规格正典。**未读过 spec-prop-library,却自发做出:判官权
   全部外包 ask 投票、一次输赢不判、笑按钮变胜负条件——与 §1.4「给指称不给感知
   + ask 共识唯一通道」完全一致。**约束逼出了正典设计**,等于正典获得一次独立
   复现实验。
2. **主持要的不是 ASR,是笑声信号。**主持自述:"丢的不是内容是时机……最想要的是
   听见有没有人在笑、笑了多久;有内容没笑声,我反而会变成念稿的。"——**音频能量
   信号比全量转写便宜一个量级且更管用**。perceive 首件的形态就此定调:分贝/能量
   优先,转写降级为特定时刻可选。
3. **参与度可测,情绪不可测。**满勤投票+笑声+退出后自愿回来 = 证明"在参与",
   不证明"在快乐"。复盘口径照此收敛,不做情绪臆测指标。
4. **人的默认是张嘴,不是按键。**求助菜单零使用;主持**预判问题、提前放宽整类
   规则**的做法成功——**宽默认扛得住聋,求助按钮扛不住**。产品交互原则:
   少做按钮,多做宽默认。

## 三、对表点一处(合并前处理)

「ask 限时问询:没人答续窗继续催不往下走」——**续窗不结算 = 对**(反虚构在结算上
的推论:无共识不得造共识);**"继续催"须服从既有铁律"同一件事不许连续催促超过
一次"**——正确形态为窗口静默续期(系统不结算不推进),语言不重复催。合并时对齐。
