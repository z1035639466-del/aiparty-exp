# 狂野模式（活局长）· 立项（2026-07-15 · 双模定案 §31，与普通模式并行）

> 定位：LLM 全程主持的现场模式（Mode B），与预编译模式（Mode A）并行双模。
> 原则：结构不删除，从编译期搬到运行时——**六原语即工具 API，封闭闩升级为 API 事实**。

## Mode B 骨架

- **工具面**：show / ask / random / timer / state / fx 六件，模型逐回合调用；
  账本写入仅经工具（僭越在接口层物理不可能）。加一件薄垫：记分板 state pad
  （score/round/holder 三键起步）。
- **System prompt**：skill v2.1.1 的铁律段 + 口味滑块钳制 + 判定三层与房主修正权 +
  感知时刻化条款——教学示例段不带（活局不写 JSON）。
- **上下文佐料**：本桌素材 + 手拌新采原子 10–20 条（不等建库，raw 直接进锅）。
- **节奏**：模型每回合输出 ≤3 句话 + ≤2 个工具调用；流式播报。

## 试玩协议（定夺一晚）

同一张三人桌：**局一四绿件（Mode A 基线，已打）vs 活局长一晚（Mode B）**。
只记四个二值 + 一句话：完局？卡壳（模型忘分/前后矛盾）几次？哪个瞬间比脚本局
更活（现挂/救场/读场）？还想再来哪种？——桌子说了算。

## 封存与沿用

沿用：六原语·铁律·滑块·账本·判定三层·修正权·素材/原子·感知时刻化·额度制
（活局=浓局档乘数）。
封存待命（Mode A 资产，不拆）：编译器·params 硬闸·判级裁判（转任局后复盘员）·
组装层过滤（活局由在场选择替代）。
评测转轨：episode 留痕（工具调用流水=天然日志）+ 局后复盘 + 真人局四问。

## 风险照实（不设门，只记账）

单局 $0.3–0.6 · 回合延迟 1–3s · 长局状态漂移（state pad 缓解）· 评测重启成本。
