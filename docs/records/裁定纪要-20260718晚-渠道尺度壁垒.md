# 裁定纪要 · 2026-07-18 晚(渠道/尺度/壁垒/模仿判分/测试设备)

> 房主口裁,Fable 落档。大方向待议清单 ①②③ 结案,④(门槛验收单)仍悬。

## ① 上桌形态与渠道(结案)

- **全员入网(多机私显)= 产品前提**,不做一机公放降级局。
- **载体 = 通用 App,全球上架**(App Store + Google Play);**小程序否决**;
  浏览器不作用户端(体验+工作量),仅作开发驾驶舱(见 §5)。
- 实施注:跨端框架一套代码出双端;dev 模式 web build 充当六机模拟台。
- **浏览器损失上报纪律(房主补令)**:写代码期间凡因兼容浏览器导致任何设计/
  体验损失,**逐件上报房主权衡**,由房主决定是否彻底丢弃浏览器端;默认取向
  = App 体验优先("东西够好玩,下载不是门槛,下载量反而助推广;连下都不愿
  下,浏览器也救不了")。推论:开发驾驶舱 web build 不受此限——它不是产品,
  **不为它妥协任何产品设计**;该纪律真正约束的是未来若考虑用户侧 Web 时的
  每一次取舍。

## ② 内容尺度(结案)

- **先按全球宽尺度做,不预设闸门**。中国等低尺度地区 = 上线后加地区闸,
  属运营期调参,不属开发期设计。"不要自己臆想给自己加一堆不必要的限制"(房主原话)。
- 底线仅保留:skill 铁律安全条款(拒绝权/非羞辱)+ S 位结构性危害位——这是
  玩法质量件不是合规件,不随地区变。
- 商店现实核对:App Store/Google Play 对饮酒内容只要求**年龄分级**(17+/18+),
  不要求内容阉割——全球宽尺度与上架兼容,无需自阉。

## ③ 壁垒与飞轮(结案)

- **壁垒 = 原子库**:筛选与结构化方法外部不可见 + 复制需时间 + 生态位先占
  (等对手研究明白,生态位已占、软件已卖)。
- **飞轮照做,范围按房主定义**:用真实 episode 数据迭代——agent 行为方式 /
  数据处理方式 / API 调用方式 / 原语设置。埋点(笑声/跳过/时长/复玩)自 M1
  第一行代码起带上。

## ④ 模仿判分(抖音链接→模仿→相似度打分)三层设计

1. **取原片层(尽力而为)**:链接解析抓帧,成败皆可;失败降级为"原片玩家
   手机自放,系统只收模仿视频"。不押注抖音解析稳定性,ToS 风险记账。
2. **判分层(perceive E3 组件)**:多模态模型抽帧对比,输出判词+分数——
   目标是"好笑且可仲裁",不是运动科学;MediaPipe 姿态量化版后置不进 MVP。
3. **兜底层(既有纪律)**:置信度低/玩家不服 → 举报 tap+共识仲裁;
   判分是节目效果,账本只认 state 写入。

## ⑤ 测试设备矩阵(现有硬件即够)

Mac:多 iOS 模拟器实例 + Android AVD;Windows:Android 模拟器 ×2;
真机:iPhone ×2 + Android ×1。六端齐,双系统真机触感在内。
六机模拟台跑 dev web build,浏览器仅此用途。

## ⑥ 命名工单(开;2026-07-18 房主二次收窄标准)

要求(房主):短、酷、迎合青年、全球传播性、先入为主的品类占位感
(TikTok/X/Snap 级);**最入脑、有节奏有特色、"20 岁的感觉,不粘一点中年味"**。
推广定式:AI 视频 + 产品现实场景实拍,**名字要能当音频钩子用**(可喊、可打拍、
可循环——TikTok 的 t-k/t-k 爆破音节奏即范本)→ 叠音爆破结构优先。
首批候选中 Kiki(k-k 叠音)/ JuJu(j-j 叠音,源自圈层词「局」)最合此标;
Rager 留作野系备选;Wildr(去元音拼法=2010s 陈旧感)、Romp/Hoot(偏温和/
中年味)降权。候选与商标/域名/handle 尽调另行工单,定名前占位名不进任何
用户可见面。
