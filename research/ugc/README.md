# UGC 角色词与查询种子

这里存放“局头/带局人/氛围担当”发现阶段的可复查输入与日志。目标不是让 AI 凭翻译猜黑话，而是把每个候选词放回本地语境，用公开网页先核对含义，再去已登录的平台验证它是否真的有人搜、有人用、能否持续找到创作者和玩法。

## 文件

- `live/collection_summary_2026-07-16.md`：本轮数字、核心答案、文件索引与研究空白的总入口。
- `query_seeds.json`：中文、韩文、英文、西班牙文、意大利文、法文、葡萄牙文、德文、日文、菲律宾和印度场景的三轴词表，并补充必要的游戏锚点、首轮查询和公开来源。
- `p0_query_log.csv`：平台原生验证日志。每搜索一次写一行，保留自动补全、相关词、标签、创作者、结果抽样和登录结果。
- `atoms.jsonl`：已人工看过原帖、完成改写、来源与安全标注的机制原子库；不复制原帖长文案。
- `live/xhs_discovery_2026-07-16.md`：小红书“局头”词河、二次回搜和风险词发现。
- `live/douyin_discovery_2026-07-16.md`：抖音“通天代/带、前摇、从夯到拉、锐评”等黑话证据。
- `live/tiktok_ads_discovery_2026-07-16.md`：TikTok ADS 登录态中的 pregame、提示卡、命名玩法和主持词。
- `live/z_gen_slang_crosswalk_2026-07-16.md`：中外功能对照；明确它不是逐字翻译表。
- `live/overseas_public_discovery_2026-07-16.md`：多语种公开网页、YouTube、Reddit、应用商店与规则站的补充证据。
- `live/batch_probe_run_2026-07-16.md`：17 个小体量批量探针的实跑结果；大 dump 与 torrent payload 保持禁用。

## 状态怎么读

- `verified`：公开网页已经把词与目标场合、角色、游戏或罚则直接连在一起。这只是“语义已核”，不代表小红书/TikTok 上量大。
- `candidate`：有相邻或专业服务语境证据，但还不能确认是真人 UGC 的活标签。
- `ambiguous`：存在强烈的其他含义，或它其实属于相邻轴。不能单独搜，只能与已验证的场合词、角色词或游戏词组合。

状态只允许通过平台证据升级：先记录搜索框自动补全，再人工检查至少 20 条结果；目标语境持续共现、而且能顺藤摸到创作者或玩法，才从 `candidate/ambiguous` 升为 `verified`。

`live/` 报告中的“强/verified”表示本轮可见结果已经反复证明该词或格式真实存在；它不自动改写 `query_seeds.json`。只有 `p0_query_log.csv` 明确记录了足够的人工样本和重复目标语境后，才把词晋升进正式批量种子。样本数没有可靠记录时保持为空，不用估算值填表。

## 采集顺序

1. 按 `场合词 × 角色词 ×（游戏锚点或罚则词）` 生成查询，先跑每个语区的 P0。
2. 记录搜索框自动补全、“大家还在搜”、相关标签；查看前 20 条并统计真正相关的数量。
3. 抽取高频新词作为下一轮种子，再反查高产创作者主页、系列标题和评论区变体。
4. 已登录页面失败、挑战验证、区域墙和 ADS/Chrome 登录态都必须写入 `access_outcome`，不能把“没访问到”误记成“没有结果”。
5. TikTok 当前普通登录不可用时，可把同一查询交给 ADS 浏览器登录态；日志中的 `session_surface` 写 `ads_browser`，`login_state` 写实际状态。

## 容易走偏的词

- 中文 `局`、`带节奏的` 太宽；`NPC` 有大量游戏语境。
- 韩文 `권주가` 是劝酒歌，也可能作为罚则 BGM，不是通用罚则词。
- 英文 `function`、`ringleader`、`instigator` 多义或偏负面；`social chair` 主要限定美国兄弟会/姐妹会。
- 意大利文 `mazziere` 是发牌人，只适合卡牌玩法中的局部主持角色。
- 葡萄牙文 `o puxador` 常指球迷看台领喊者；`prenda` 常指礼物。
- 菲律宾语境里 `tagay` 是敬酒/轮饮动作，`barkada` 是朋友圈，`walwal` 是豪饮状态，都不是罚则；`tanggero` 是倒酒/轮次协助者。
- 印度 `anchor` 必须与 `party/Tambola/kitty` 连用，避免新闻主播噪声；`Tambola caller` 才是高精度角色词。

## 原子入库纪律

不复制原帖长段文案、字幕或评论。只保留自行改写的机制事实、必要的短标签、原始 URL、语区、角色类型、玩法/罚则和证据状态。涉及强制饮酒时必须同时写非酒替代或档位钩；涉及亲密、羞辱或成人内容时必须写可拒绝机制和成人分级。无法验证原内容时，只留在查询日志，不写入 `atoms.jsonl`。

`atoms.jsonl` 每行必须是一个独立 JSON 对象，空行会被忽略。字段以采集程序的当前 schema 为准；不要在 JSONL 里放注释或数组外壳。
